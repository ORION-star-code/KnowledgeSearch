# 2026.4.14-配置文档

## 1. 适用场景

本文档适用于当前项目的服务器部署场景：

- MySQL、Elasticsearch、Kibana、Canal 都部署在服务器上的 Docker Compose 中
- Spring Boot 应用可以运行在服务器宿主机上，或者继续容器化部署
- 当前项目默认架构仍然是：

`MySQL -> Canal -> Sync Worker -> Elasticsearch`

本文档中的配置可以直接复制使用。

---

## 2. docker-compose.yml

文件路径建议：`/opt/KnowledgeSearch/docker-compose.yml`

```yaml
version: "3.9"

services:
  mysql:
    image: mysql:8.0
    container_name: knowledge-search-mysql
    restart: unless-stopped
    command:
      - --default-authentication-plugin=mysql_native_password
      - --character-set-server=utf8mb4
      - --collation-server=utf8mb4_unicode_ci
      - --server-id=1
      - --log-bin=mysql-bin
      - --binlog_format=ROW
      - --binlog_row_image=FULL
      - --expire_logs_days=7
      - --binlog-do-db=knowledge_search
    environment:
      MYSQL_ROOT_PASSWORD: 123456
      MYSQL_DATABASE: knowledge_search
      TZ: Asia/Shanghai
    ports:
      - "3306:3306"
    volumes:
      - ./deploy/mysql/data:/var/lib/mysql
      - ./deploy/mysql/init:/docker-entrypoint-initdb.d
    networks:
      - knowledge-search-net

  elasticsearch:
    image: docker.elastic.co/elasticsearch/elasticsearch:8.18.0
    container_name: knowledge-search-es
    restart: unless-stopped
    environment:
      discovery.type: single-node
      xpack.security.enabled: "false"
      ES_JAVA_OPTS: -Xms512m -Xmx512m
      TZ: Asia/Shanghai
    ports:
      - "9200:9200"
      - "9300:9300"
    volumes:
      - ./deploy/elasticsearch/data:/usr/share/elasticsearch/data
    networks:
      - knowledge-search-net

  kibana:
    image: docker.elastic.co/kibana/kibana:8.18.0
    container_name: knowledge-search-kibana
    restart: unless-stopped
    depends_on:
      - elasticsearch
    environment:
      ELASTICSEARCH_HOSTS: "http://elasticsearch:9200"
      I18N_LOCALE: "zh-CN"
      TZ: Asia/Shanghai
    ports:
      - "5601:5601"
    networks:
      - knowledge-search-net

  canal-server:
    image: canal/canal-server:v1.1.7
    container_name: knowledge-search-canal
    restart: unless-stopped
    depends_on:
      - mysql
    environment:
      canal.destinations: example
      canal.instance.tsdb.enable: "true"
      canal.instance.memory.buffer.size: "16384"
      canal.instance.global.spring.xml: classpath:spring/file-instance.xml
      TZ: Asia/Shanghai
    ports:
      - "11111:11111"
    volumes:
      - ./deploy/canal/conf/example/instance.properties:/home/admin/canal-server/conf/example/instance.properties
      - ./deploy/canal/logs:/home/admin/canal-server/logs
    networks:
      - knowledge-search-net

networks:
  knowledge-search-net:
    driver: bridge
```

## 3. Canal 实例配置

文件路径：
`deploy/canal/conf/example/instance.properties`

```properties
canal.instance.mysql.slaveId=1234

canal.instance.master.address=knowledge-search-mysql:3306
canal.instance.dbUsername=canal
canal.instance.dbPassword=canal123456
canal.instance.connectionCharset=UTF-8

canal.instance.tsdb.enable=true
canal.instance.gtidon=false

canal.instance.filter.regex=knowledge_search\\.(kb_article|kb_category|kb_tag|kb_article_tag)
canal.instance.filter.black.regex=

canal.instance.detecting.enable=true
canal.instance.detecting.sql=SELECT 1
```

说明：

- `mysql:3306` 是正确的，因为 Canal 和 MySQL 在同一个 Docker Compose 网络中
- `example` 必须和应用配置中的 `app.canal.destination=example` 保持一致
- 监听表范围与当前项目同步设计一致：
  - `kb_article`
  - `kb_category`
  - `kb_tag`
  - `kb_article_tag`

## 4. MySQL Canal 账号初始化 SQL

文件路径：
`deploy/mysql/init/02-canal-user.sql`

```sql
CREATE USER IF NOT EXISTS 'canal'@'%' IDENTIFIED BY 'canal123456';
GRANT SELECT, REPLICATION SLAVE, REPLICATION CLIENT ON *.* TO 'canal'@'%';
FLUSH PRIVILEGES;
```

如果数据库已经初始化过一次，而这个 SQL 没有自动执行，需要手动进入 MySQL 执行一次。

## 5. Spring Boot 应用配置

下面分两种情况。

### 5.1 应用运行在服务器宿主机上

如果你的 Spring Boot 应用不是容器运行，而是直接在服务器宿主机上运行，那么推荐这样配：

```yaml
spring:
  datasource:
    url: jdbc:mysql://127.0.0.1:3306/knowledge_search?useUnicode=true&characterEncoding=utf8&serverTimezone=Asia/Shanghai
    username: root
    password: 123456
    driver-class-name: com.mysql.cj.jdbc.Driver
  elasticsearch:
    uris: http://127.0.0.1:9200

app:
  canal:
    enabled: true
    destination: example
    server: 127.0.0.1:11111
    username: ""
    password: ""
```

如果不是同机部署，而是应用部署在另一台机器上，就把上面的 `127.0.0.1` 改成 Docker 服务器的真实 IP。

例如：

```yaml
spring:
  datasource:
    url: jdbc:mysql://192.168.1.100:3306/knowledge_search?useUnicode=true&characterEncoding=utf8&serverTimezone=Asia/Shanghai
    username: root
    password: 123456
    driver-class-name: com.mysql.cj.jdbc.Driver
  elasticsearch:
    uris: http://192.168.1.100:9200

app:
  canal:
    enabled: true
    destination: example
    server: 192.168.1.100:11111
    username: ""
    password: ""
```

### 5.2 应用也运行在同一个 Docker Compose 网络里

如果你的 Spring Boot 应用后面也会容器化，并加入同一个 `knowledge-search-net` 网络，那么应用配置应改为服务名访问：

```yaml
spring:
  datasource:
    url: jdbc:mysql://mysql:3306/knowledge_search?useUnicode=true&characterEncoding=utf8&serverTimezone=Asia/Shanghai
    username: root
    password: 123456
    driver-class-name: com.mysql.cj.jdbc.Driver
  elasticsearch:
    uris: http://elasticsearch:9200

app:
  canal:
    enabled: true
    destination: example
    server: canal-server:11111
    username: ""
    password: ""
```

## 7. 验证项

### 7.1 验证 MySQL binlog

进入 MySQL：

```bash
docker exec -it knowledge-search-mysql mysql -uroot -p123456
```

执行：

```sql
SHOW VARIABLES LIKE 'log_bin';
SHOW VARIABLES LIKE 'binlog_format';
SHOW VARIABLES LIKE 'binlog_row_image';
SHOW MASTER STATUS;
SELECT user, host FROM mysql.user WHERE user = 'canal';
SHOW GRANTS FOR 'canal'@'%';
```

期望结果：

- `log_bin = ON`
- `binlog_format = ROW`
- `binlog_row_image = FULL`
- 存在 `canal` 用户

### 7.2 验证 Canal

查看 Canal 日志：

```bash
docker compose logs -f canal-server
```

重点确认没有以下错误：

- Canal 无法连接 `mysql:3306`
- `example` 实例启动失败
- 无法读取 binlog

如果配置正常，Canal 会加载 `example` 实例并连接到 `mysql:3306`。

### 7.3 验证 Elasticsearch

```bash
curl http://127.0.0.1:9200
curl http://127.0.0.1:9200/_cluster/health
```

### 7.4 验证 Kibana

浏览器访问：

```text
http://服务器IP:5601
```

## 8. 当前项目限制说明

这份配置可以保证以下内容正确：

- MySQL 正常开启 binlog
- Canal 能连接 MySQL 并订阅指定表
- Elasticsearch 和 Kibana 能正常启动
- Spring Boot 应用能按目标地址连接 MySQL、ES、Canal

但需要明确：

- 当前项目里的 `CanalEventAdapterImpl` 还是占位实现
- 当前仓库还没有完成真实 Canal 原始消息解析
- 所以“Canal 基础设施配置正确”不等于“项目已经打通真实增量同步到 ES”

也就是说，基础设施层没问题后，后续还需要继续开发真实的 Canal 消费与事件解析逻辑。
