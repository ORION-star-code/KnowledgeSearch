# 2026.4.15 搜索改进

## 1. 改动背景

本次改动基于 `2026.4.15-改进思路.md` 中搜索优化方向，优先落地对用户体验最直接的部分：

- 将高亮逻辑从 Service 层字符串替换改为由搜索仓储返回高亮结果
- 补齐 V1 范围内常用搜索过滤条件
- 收敛排序策略，提升无关键字和有关键字两类场景下的结果质量
- 同步更新接口文档与测试，保证行为可验证

---

## 2. 本次完成内容

### 2.1 高亮逻辑改造

原实现中，`KnowledgeSearchServiceImpl` 通过 `String.replace(...)` 直接替换关键字生成高亮片段，这种方式存在明显问题：

- 不能反映 Elasticsearch 真实命中片段
- 对大小写与命中位置处理不稳定
- 容易误替换非真实命中内容

本次调整后：

- Elasticsearch 仓储使用原生 highlight 返回 `title`、`summary`、`content` 的高亮字段
- Service 层只负责结果映射，不再自行拼接高亮内容
- 测试环境的内存仓储也统一返回高亮字段结构，保证测试语义与生产实现一致

### 2.2 搜索过滤条件增强

在原有 `keyword`、`categoryId`、`tagNames`、`status` 的基础上，新增：

- `author`
- `publishTimeStart`
- `publishTimeEnd`
- `updatedTimeStart`
- `updatedTimeEnd`

对应搜索查询对象和仓储过滤逻辑已同步补齐。

### 2.3 默认搜索行为收敛

公开搜索接口 `/search` 现在在未显式传入 `status` 时，默认只查询 `published` 状态文章。

这样可以避免前台搜索误返回草稿或下线内容，更符合当前系统公开搜索场景。

### 2.4 排序策略优化

本次对排序行为做了以下收敛：

- 无关键字时，默认按 `publishTime -> updatedAt -> id` 倒序
- 有关键字且非 `latest` 排序时，按相关性优先，再按时间排序
- 对标题完全命中和标题前缀命中增加额外权重

这样可以提升：

- 列表浏览场景下的“最近内容优先”
- 关键词检索场景下的“标题更相关结果优先”

### 2.5 文档与注解修复

本次改动范围内同步修复了搜索接口相关 Swagger/OpenAPI 注解中的中文乱码问题，包括：

- `SearchController`
- `SearchRequest`
- `SearchResponse`
- `SearchResultItem`

## 4. 兼容性说明

### 4.1 接口兼容性

本次对 `/search` 接口属于向后兼容扩展：

- 新增查询参数不会破坏旧调用
- 原返回结构未移除字段

### 4.2 行为变化

有一个需要注意的行为变化：

- 当未传 `status` 时，默认只返回 `published` 文章

如果后续存在后台搜索或管理端复用该接口，并且需要查看 `draft` / `offline` 内容，需要显式传入状态参数，或在后续拆分后台检索接口。

### 4.3 内部契约变化

`ArticleIndexRepository.search(...)` 的返回类型从单纯文档分页改为带高亮结果的分页对象。这属于内部搜索仓储契约调整，不影响对外 HTTP API，但后续如果有其他模块直接依赖该仓储，需要注意适配。

## 6. 当前结果与后续建议

### 6.1 当前结果

本次改动已经完成以下目标：

- 高亮能力从演示型实现升级为更贴近真实搜索系统的实现
- 搜索过滤维度更完整
- 默认公开搜索行为更安全
- 排序策略更符合用户直觉
- 接口文档和测试与实现保持一致

### 6.2 建议的下一步

后续可继续按优先级推进：

1. 进一步优化 ES 查询权重与高亮片段策略
2. 增加 repository 级查询组装单元测试
3. 视需要补充后台检索场景与公开检索场景的接口分离
4. 在后续涉及 mapping/analyzer 调整时，再设计索引版本化与 alias 切换方案
